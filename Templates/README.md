# Templates

The **Templates** folder contains reusable documentation structures used throughout the **Root & Repository** project.

Each template represents an established workflow that has been tested through real labs, investigations, projects, or learning activities. Templates are added only after the documentation format has become consistent and repeatable.

---

# Purpose

The purpose of this folder is to:

* Maintain consistent documentation across the repository
* Reduce repetitive formatting work
* Preserve established documentation standards
* Support scalable repository growth
* Provide a reliable starting point for new work

---

# Template Standards

Every template in this folder should:

* Reflect an actively used workflow
* Preserve the established section order
* Include clear placeholders for new content
* Remain free of room-specific, project-specific, or investigation-specific information
* Match the documentation standards used throughout the repository
* Be updated only when the associated workflow is intentionally revised

Templates should not be created for workflows that are still experimental or incomplete.

# Usage

When beginning new documentation:

1. Select the appropriate template.
2. Copy it into the correct repository location.
3. Rename the copied file according to the established folder structure.
4. Replace all placeholders with information from the current activity.
5. Preserve the template's section order and documentation standards.
6. Keep the original template unchanged.

---

# Template Management

The files in this folder serve as the master reusable versions.

Completed documentation should remain in its appropriate project, investigation, module, or room folder. Templates should not contain completed activity content or be modified for one-time use.

Any change to a locked workflow must be made intentionally and reflected in both the template and its related documentation standards.

---

> A template belongs in this folder only when it represents the exact format consistently used throughout the **Root & Repository** project.
