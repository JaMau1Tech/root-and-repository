# Investigations

This directory contains structured technical investigations completed while developing troubleshooting, analytical thinking, documentation, and cybersecurity skills.

Each investigation records the original problem, commands and tools used, evidence collected, findings, root cause, resolution, lessons learned, and supporting screenshots.

---

# Investigation Categories

## Windows

Windows troubleshooting and system investigations involving operating system behavior, configuration issues, commands, services, files, and administrative tools.

## Wireshark

Network traffic investigations involving packet captures, protocols, filters, communication patterns, and observed network behavior.

# Investigation Workflow

Every investigation follows the locked Root & Repository investigation workflow:

1. Define the problem
2. Record the environment
3. Document commands and tools
4. Investigate methodically
5. Record evidence and findings
6. Identify the root cause
7. Apply and verify the resolution
8. Document lessons learned
9. Capture supporting screenshots
10. Produce professional documentation
