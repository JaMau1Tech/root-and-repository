# Cheat Sheets

This folder contains concise, original reference materials designed for quick review during labs, troubleshooting, studying, and technical practice.

Cheat sheets summarize concepts but do not replace deeper learning or hands-on experience.

---

## Purpose

- Provide quick command references
- Reinforce frequently used concepts
- Support troubleshooting
- Prepare for assessments and interviews
- Reduce time spent searching for basic syntax
- Connect theory to practical use

---

## Planned Cheat Sheets

- Linux command reference
- Windows command reference
- PowerShell basics
- Bash basics
- Git and GitHub commands
- Networking fundamentals
- Common ports and protocols
- OSI model
- TCP/IP model
- DNS and DHCP
- Nmap commands
- Wireshark display filters
- Binary and hexadecimal conversions
- File permissions
- Help Desk troubleshooting workflow

---

## Cheat Sheet Standard

Each cheat sheet should include:

1. Topic name
2. Purpose
3. Key terms
4. Commands or syntax
5. Short examples
6. Common mistakes
7. Security or troubleshooting relevance

---

## Naming Convention

Use lowercase, descriptive, hyphen-separated filenames.

Examples:

```text
linux-command-cheat-sheet.md
common-ports-and-protocols.md
git-command-reference.md
binary-hex-conversion-guide.md
