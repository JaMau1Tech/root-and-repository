# Root & Repository Standards Index

**Version:** 1.0  
**Status:** 🔒 LOCKED  
**Last Updated:** July 12, 2026

The **Standards** folder contains the official workflows, documentation requirements, repository structures, notebook formats, screenshot conventions, assessment standards, and career development processes used throughout the **Root & Repository** project.

These documents serve as the governance layer for the repository. They ensure consistency, preserve approved workflows, and prevent documentation or process drift as the repository continues to grow.

---

# Guiding Principles

- Consistency is more important than speed.
- Every workflow must be followed in its documented order.
- Documentation must be accurate, reproducible, and professional.
- Hands-on learning must be supported by structured documentation.
- Repository organization must remain consistent across all projects.
- Templates must reflect approved and actively used workflows.
- Workflow improvements require explicit approval before implementation.
- Approved changes must be documented and versioned.
- Locked standards must not be changed, reordered, renamed, expanded, or reduced without explicit approval.

## TryHackMe

### `TryHackMe-Workflow.md`

Defines the official workflow for completing, documenting, organizing, and committing standard TryHackMe rooms.

This workflow includes:

- Task explanations
- Handwritten notebook notes
- Question completion
- Task screenshots
- Hands-on lab documentation
- Lab screenshots
- Room-completion evidence
- Module documentation
- Repository updates
- Git commands

Topic Transition Recap rooms follow the separate recap-only process defined within the approved TryHackMe standards.
