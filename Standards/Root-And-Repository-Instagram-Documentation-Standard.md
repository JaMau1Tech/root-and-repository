**Version:** 1.0  
**Status:** 🔒 Locked

# Purpose

The Root & Repository Instagram serves as the **visual companion** to my GitHub portfolio.

Its purpose is to professionally document my journey toward becoming an **IT Support Professional** through enterprise home lab projects, certifications, networking, Linux, Windows Server, Active Directory, cybersecurity, and technical documentation.

Instagram tells the story.

GitHub contains the complete technical documentation.

The two platforms should always complement one another.

---

# Core Principles

- Document authentic milestones.
- Never exaggerate progress.
- Never imply a project was completed "today" unless it actually was.
- If sharing an older project, present it as:

> "I'm sharing another documented milestone from my IT journey..."

- Keep every post educational.
- Show troubleshooting and lessons learned.
- Focus on the learning process rather than only the finished result.
- Every project should naturally lead into the next.

---

# Root & Repository Log Naming Convention

Every post begins with:

```text
🖥️ Root & Repository Log #00X | Project Title
```

Examples:

```text
🖥️ Root & Repository Log #001 | Welcome to My IT Journey

🖥️ Root & Repository Log #002 | Building My Enterprise Home Lab

🐧 Root & Repository Log #003 | Installing My First Ubuntu Virtual Machine

🪟 Root & Repository Log #004 | Installing Windows Server 2022
```

---

# Instagram Caption Standards

Every caption should include:

## 1. Introduction

- Explain what project is being documented.
- If applicable, acknowledge that it is a milestone being shared from an existing learning journey.

Example:

> Today I'm sharing another documented milestone from my IT journey...

---

## 2. Story

Explain:

- Why the project matters
- Why it was performed
- Previous experience (if relevant)
- Obstacles encountered
- Decisions made
- Troubleshooting completed

The story should feel authentic and chronological.

---

## 3. Technical Summary

When appropriate, include:

- Hardware
- Software
- Technologies
- Operating Systems
- Virtualization Platforms

---

## 4. Skills Learned

Use concise bullet points.

Example:

- 🐧 Ubuntu Linux
- 💿 VMware Workstation Pro
- ⚙️ Virtualization
- 🔍 Troubleshooting
- 📝 Technical Documentation

---

## 5. Reflection

Briefly explain:

- What was learned
- Why it matters
- How it contributes toward becoming an IT Support Professional

---

## 6. Closing

Every post ends with:

> 📁 Every project I share is part of my journey toward becoming an IT Support professional. Thanks for following along as I continue building, learning, and documenting one project at a time.
>
> 🚀 Root & Repository

---

# Caption Guidelines

Every caption should be:

- Professional
- Authentic
- Educational
- Conversational
- Under Instagram's 2,200-character limit

Avoid:

- Clickbait
- Overly dramatic language
- Exaggerated accomplishments
- Misleading timelines

---

# Troubleshooting Standard

Whenever troubleshooting occurred, explain:

- The problem
- The investigation
- The solution
- What was learned

Example:

> Virtualization initially failed because...
>
> After troubleshooting...
>
> The issue was resolved by...

Never hide challenges.

Troubleshooting demonstrates growth.

# Alt Text Standard

Every image receives unique accessibility text.

Example:

> Ubuntu Linux running successfully inside VMware Workstation Pro on my enterprise home lab.

Alt text should:

- Describe what is visible
- Mention important technologies
- Be concise
- Improve accessibility

# Root & Repository Philosophy

Every post should answer four questions:

1. What did I build?
2. Why did I build it?
3. What did I learn?
4. What comes next?

The goal is not to impress people with finished projects.

The goal is to professionally document continuous growth into an IT Support career.
